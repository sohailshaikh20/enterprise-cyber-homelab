# Contributing

Thanks for wanting to improve the lab docs or scripts.

## How to contribute
1. Fork the repo
2. Create a feature branch
3. Make your changes with clear commits
4. Open a Pull Request

## Documentation guidelines
- Prefer short sections with headings
- Include screenshots only if they add signal
- Never commit secrets or real credentials

## Issue labels
- `bug` — broken docs/scripts
- `enhancement` — improvements or additions
- `security` — anything that could cause unsafe use or exposure
